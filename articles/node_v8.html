Node and V8





What is V8?

V8 is a JavaScript (JS) engine. A JS engine is merely a program that compiles code at runtime. It was written in C++ and released by Google in 2008 and remains as Google Chome’s JavaScript engine (every major browser uses a JS engine. For example, Firefox uses SpiderMonkey). And it’s very powerful – it allows the web to run massive amounts of JS code. This engine is constantly being improved so that the web experience can go faster.



A way that V8 makes the web very interesting is how you can add capabilities to your JS with C++. JS is limited with what it can do, but you can add C++ to expand its features since the engine breaks down the language into machine code instead of interpreting it.



What is Node?

Node is a standalone JS runtime environment used for server-side applications. It allows users to run JS in the V8 engine without needing a web browser. It makes applications easily scalable and more efficient, like using modules as discussed in a previous video. And V8 is the engine that runs Node! These two programs are very tightly coupled, meaning that Node relies on V8 very heavily. Node.js is important to learn since it is used by many companies and has become a big part of the web development stack.
